# Example Project AGENTS

Use this file in a project repository when you want project-specific rules on top of the global `codex-solo-kit` rules.

## Project Goal

Describe what this product does, who it serves, and what success looks like.

## Architecture Notes

- List the key modules and responsibilities.
- Call out state management, networking, storage, and platform boundaries.

## Coding Expectations

- Preserve the existing architecture unless there is a strong reason to change it.
- Prefer incremental changes over broad rewrites.
- Keep platform-specific behavior isolated.

## Verification

- List the commands that should be run after code changes.
- List the release-critical flows that need manual checking.
